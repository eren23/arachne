Engine Unlock: Custom Sprites via Embedded PNG

The engine has load_png_bytes() — documented as "Useful for WASM where textures are fetched as byte arrays." We can embed pixel art PNGs as const byte arrays in Rust, load them at startup, and use them as sprite textures.

Size impact: A 32x32 pixel art PNG = ~200-500 bytes. Even 30 sprites = ~10-15KB. The WASM binary is 389KB — this adds ~3% and unlocks a completely different visual quality.

This means: actual character sprites, environmental objects, NPC portraits, UI elements — not just colored rectangles.

Revised Design: What Changes

1. Dialogue System Overhaul

RPG-style dialogue box at bottom of screen with speaker name + portrait. Not just floating text.

Arachne
You found the web. Good. Most people just scroll past. Let me show you what Eren built.
Terminal: Arachne Engine
389 kilobytes. The engine running this game. Most hero images on the web weigh more than that.
Eren's Desk
M.Sc. Intelligent Systems. Berlin. Mass producer of open source. Talks to machines for a living.

2. Environmental Objects, Not Terminals

Instead of "walk to colored square, press E", you walk to actual pixel art objects:

  • The Forge: An anvil sprite for Arachne, a glowing furnace sprite for Synapse, a stone monument sprite for sqld
  • The Hive: Server rack sprites with blinking lights, terminal sprites with scrolling text effect, small robot sprites drifting around
  • The Lens: A magnifying glass on a table, a globe, monitor sprites showing "vision" patterns
  • The Lab: Bookshelves, bubbling flask sprites, scattered note sprites
  • Eren's Room: Desk with computer, bookshelf, potted plant, window showing pixel Berlin skyline

3. Spider Companion With Personality

Arachne isn't decoration — she's the narrator. With an actual spider sprite, she becomes a character:

Arachne (entering The Forge)
Careful here. This is where Eren decided colored rectangles weren't enough. So he built an engine.
Arachne (entering The Hive)
The agents in here built a database. Autonomously. 26,500 lines. Eren just watched. Terrifying, honestly.
Arachne (entering Eren's Room)
He doesn't sleep enough. Don't tell him I said that.

4. Dynamic Room Moments

  • The Hive: 3-5 small bot sprites drift around the room autonomously (simple sine-wave movement). Visual representation of the swarm.
  • The Forge: Lava tiles "pulse" (swap between lava/orange tiles on a timer). Feels alive.
  • The Void: After reading the final dialogue, web strands (dark gray tiles) fill the void — the web is complete. Tiles literally change.
  • Nexus: Each time you clear a room, a small sprite appears on that room's web strand — visual progress.

5. Proper Welcome Experience

Not just a title card. The welcome screen is a cold open:

Black screen. Pause. Then text fades in character by character (typewriter):
"389 kilobytes."
Pause.
"That's all this needs."
Pause.
"Press any key to enter the web."

Spider web tiles materialize from center outward. Then the Nexus loads. First impression = mystery + flex.

Decision Points

  1. Sprite generation approach: Use Nano Banana Pro to batch-generate pixel art. Top-down 16x16/32x32, consistent RPG style. Eren generates, we embed as byte arrays.
  2. How many sprites: ~20-30 unique sprites total. 3 characters + 10-15 objects + 5-8 UI/symbols. ~15KB total embedded size.
  3. Directional player sprite: 4 directions (N/S/E/W) or single forward-facing? 4 directions = more polished but 4x sprites for player.
  4. Dialogue box style: Full RPG box with 9-slice border + speaker name? Or simpler bottom-bar with text? RPG box is more impressive but needs a border texture.
  5. Tile grids: Keep the grids I designed (with adjustments) — they work for layout. The sprites go ON TOP of the tile grid as entities. Tiles = floor/walls, sprites = objects/characters.
